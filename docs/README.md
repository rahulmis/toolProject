## üîÑ Documentation Maintenance

### Adding New Documentation:
1. Create `.md` file in this `docs/` folder
2. Use clear, descriptive filename (e.g., `FEATURE_NAME_GUIDE.md`)
3. Add entry to this `README.md` index
4. Keep feature-specific docs in `src/features/[feature-name]/README.md`

### Documentation Standards:
- ‚úÖ Use Markdown format
- ‚úÖ Include clear headings and sections
- ‚úÖ Add table of contents for long documents
- ‚úÖ Use code examples where helpful
- ‚úÖ Keep docs up-to-date with code changes

### What Goes Where:
- **Root `/README.md`** - Project overview for GitHub/new visitors
- **`/docs/`** - All project-wide documentation
- **`/src/features/[name]/README.md`** - Feature-specific implementation docs
